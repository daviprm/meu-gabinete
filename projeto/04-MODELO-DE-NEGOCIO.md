# Modelo de Negócio

## Formato: SaaS (Software as a Service)

O Meu Gabinete opera como um **serviço por assinatura**, onde cada gabinete paga mensalmente pelo acesso à plataforma. Isso garante receita recorrente, escalabilidade e baixo custo de entrada para o cliente.

---

## Por Que SaaS?

| Aspecto | Por que faz sentido |
|---------|-------------------|
| **Receita recorrente** | Previsibilidade financeira, valuation alto |
| **Barreira de saída** | Quanto mais dados o gabinete coloca, mais difícil sair |
| **Escalável** | 1 gabinete ou 10.000 — a infraestrutura é a mesma |
| **Atualização contínua** | Todos os clientes sempre na versão mais recente |
| **Sem instalação** | Acessa pelo navegador, sem TI |
| **Custos operacionais baixos** | Cloud auto-escalável, sem servidor dedicado por cliente |

## Estrutura de Planos

### Plano Free (Grátis)
**Para quem:** Gabinetes pequenos querendo experimentar

- Até 100 cidadãos cadastrados
- 1 usuário
- Dashboard básico
- Gestão de demandas (até 50/mês)
- Sem mapa interativo
- Sem relatórios avançados
- Marca d'água "Meu Gabinete"

**Objetivo:** Porta de entrada. Criar hábito de uso. Converter para pago.

---

### Plano Pro — R$ 149/mês
**Para quem:** Gabinetes de vereadores, juízes de primeira instância

- Até 5.000 cidadãos
- Até 5 usuários
- Dashboard completo
- Demandas ilimitadas
- Mapa interativo
- Pesquisas de campo (até 10/mês)
- Relatórios em PDF
- Comunicação multicanal (WhatsApp, SMS, E-mail)
- Agenda integrada
- Suporte por e-mail

**Objetivo:** Plano principal. Onde está a massa de clientes.

---

### Plano Enterprise — R$ 499/mês
**Para quem:** Gabinetes de deputados, desembargadores, prefeituras

- Cidadãos ilimitados
- Usuários ilimitados
- Todos os módulos
- Pesquisas ilimitadas
- API para integrações
- Relatórios customizados
- Exportação Excel + PDF
- Suporte prioritário (WhatsApp)
- Onboarding dedicado
- Personalização da marca (white-label)

**Objetivo:** Ticket alto. Clientes estratégicos. Cases de sucesso.

---

### Plano Institucional — Sob consulta
**Para quem:** Tribunais, câmaras municipais, assembleias (contrato para múltiplos gabinetes)

- Licenciamento para N gabinetes
- Painel administrativo central
- Dados consolidados cross-gabinetes
- SLA garantido
- Treinamento presencial
- Integração com sistemas legados (PJe, SAPL, etc.)

**Objetivo:** Receita concentrada. Entrada via licitação ou contratação direta.

## Canais de Aquisição

1. **Indicação (boca a boca)** — Político indica político. Principal canal no início.
2. **Redes sociais** — Conteúdo sobre "gestão pública moderna" no Instagram/LinkedIn
3. **Eventos** — Congressos de vereadores, encontros da magistratura, conferências de gestão pública
4. **Parcerias** — Associações de magistrados, câmaras municipais, partidos
5. **Inside Sales** — Equipe fazendo contato direto com gabinetes (a partir do Ano 2)
6. **Licitação/Pregão** — Para contratos institucionais (Ano 2-3)

---

## Métricas-Chave (North Star)

| Métrica | O que mede |
|---------|-----------|
| **MRR** | Receita recorrente mensal |
| **Churn** | % de clientes que cancelam por mês (meta: < 5%) |
| **LTV** | Lifetime Value — quanto um cliente gera no total |
| **CAC** | Custo de aquisição de cliente |
| **LTV/CAC** | Precisa ser > 3x para ser sustentável |
| **NPS** | Satisfação do cliente (meta: > 50) |
| **DAU/MAU** | Usuários ativos diários/mensais — engajamento |

## Modelo Alternativo: White-Label para Partidos/Tribunais

Uma opção futura é licenciar a plataforma em formato **white-label**:
- O tribunal/partido coloca sua marca
- Distribui para todos os gabinetes vinculados
- Contrato único com valor alto
- Meu Gabinete opera nos bastidores

Isso multiplica a base de usuários exponencialmente sem precisar vender gabinete a gabinete.
